1. What is Requirement Analysis?
Your Answer

Requirement analysis is the detailed process of identifying and document test scenarios and test cases before the testing process begins.

Score: 8/10

Improvement

Requirement Analysis comes before writing test scenarios and test cases.

A better answer:

Requirement Analysis is the process of studying software requirements, business rules, and acceptance criteria to understand what needs to be tested and identify any missing or unclear requirements before testing begins.

Interview Tip: Mentioning acceptance criteria earns extra points because it's widely used in Agile teams.

2. What is a Test Scenario?
Your Answer

Test scenario is a well detailed summary of what will be tested during a test phase.

Score: 8.5/10

A small correction:

A Test Scenario is actually high-level, not highly detailed.

A stronger answer:

A Test Scenario is a high-level description of a feature or functionality that needs to be tested.

Example:

Verify Login
Verify Checkout
Verify Search

4. Difference Between Test Scenario and Test Case
Your Answer

Test scenario allows full instruction of what should be verified before testing while test case is the detailed process to verify test scenario.

Score: 8.5/10

Almost perfect.

A clearer version:

Test Scenario

Defines what to test.

Test Case

Explains how to test it.

This simple distinction is easy to remember and commonly used in interviews.

7. What makes a good Test Case?
Your Answer

Good case is a typical test scenario where features work like it is supposed to.

Score: 6.5/10

This is the one I'd like you to improve.

A good test case should be:

Clear
Easy to understand
Repeatable
Covers one objective
Includes expected results
Uses realistic test data

Think of it this way: another tester should be able to execute your test case without asking you any questions.
